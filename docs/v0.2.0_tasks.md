## 指令分類（定案）

| 頻率 | 指令 | 資料來源 |
|------|------|----------|
| Composite | `jack daily` | wlm + etl-failures + mcd etl-status/log/value-check |
| Composite | `jack weekly` | long-queries + stale-tables + audit |
| Composite | `jack monthly` | disk + table-health + skew |
| Daily | `check wlm` | `stl_wlm_query` + `stl_wlm_rule_action` + `stl_querytext` |
| Daily | `check etl-failures` | `stl_load_errors` |
| Daily | `mcd etl-status` | `sys.etl_entity_prof` |
| Daily | `mcd etl-log` | `sys.etl_audit_log` |
| Daily | `mcd value-check` | `sys.vw_tbl_stat_max_column_value` |
| Weekly | `check long-queries` | `sys.vw_tbl_query_log` |
| Weekly | `maintain stale-tables` | `information_schema.tables` |
| Weekly | `maintain audit` | `stl_ddltext` |
| Monthly | `check disk` | `svv_table_info` |
| Monthly | `check table-health` | `svv_table_info` |
| Monthly | `check skew` | `svv_table_info` |
| Incident | `incident locks` | `pg_locks` + `stv_sessions` |
| Incident | `incident spill` | `stl_query` + `stl_query_metrics` |
| Ad-hoc | `check deps` | `pg_proc` + `information_schema.views` |
| Ad-hoc | `mcd etl-missing` | `etl_table_config` × `etl_entity_prof` |
| Ad-hoc | `explain` | AWS Bedrock |
| Ad-hoc | `config` | env vars |
